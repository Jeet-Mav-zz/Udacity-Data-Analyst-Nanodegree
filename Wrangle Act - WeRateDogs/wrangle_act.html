Wrangle and Analyze Data¶

Introduction¶

The Project is based on a Twitter Account "WeRateDogs" that used @dog_rates as its twitter archive. WeRateDogs rates people's dogs with a humorous comment about the dog. These ratings are always out of 10 and the numerators are always above 10. Because why not?! Dogs are pretty cute and loyal and caring and the list goes on.

Goal¶

Since this is a real world data which rarely comes clean. My goal is to wrangle through this datasets and try creating interesting analyses and visualizations.

DataSet Used¶

  • twitter-archive-enhanced.csv

    This Dataset contains more than 5000+ tweets

Dataset contains the following columns:

  1. tweet_id
  2. in_reply_to_status_id
  3. in_reply_to_user_id
  4. timestamp
  5. source
  6. text
  7. retweeted_status_id
  8. retweeted_status_user_id
  9. retweeted_status_timestamp
  10. expanded_urls
  11. rating_numerator
  12. rating_denominator
  13. name
  14. doggo
  15. floofer
  16. pupper
  17. puppo

--

  • Image-predictions : It is a table full of image predictions along with tweet id, image URL and image number processed using neural network that classifies breeds of dogs

--

  • Tweepy (Twitter API) Tweepy is a Python library for accessing the Twitter API. It is great for simple automation and creating twitter bots.

  3. Multiple cases where numerator & denominator less than 10. These entries need to be removed.
  4. Dogs name are inconsistent
  5. There are no missing values in image dataframe
  6. No duplicated Entries
  7. Datatype of tweet_id to object and of timestamp to standard datetime
  8. Changing column names to more appropriate names.
  9. Change p1, p2, p3 to prediction1_breed, prediction2_breed, prediction3_breed respectively

  1. Merging Datasets to one single dataset
